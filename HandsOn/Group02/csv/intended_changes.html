Proposed Changes

These are changes that we thought would be necessary after analyzing the csv:

  1. YEAR
    • Every entry has a year
    • Consistent years (Ex: 200015 is not valid)
    • Two digits (ex: go from 1 (January) to 01)
    NOTE: maybe it is interesting to have an extra column with year and month like 2017-01
  2. ID
    • There are many buildings that share an id, and those that do not have an id for now we have put (blank), since it is not decided if it is a relevant column such as consumption or the year./li>
    • None of the IDs is repeated (If it is repeated check if it is correct in order to delete the row or modify it)
  3. TYPE OF BUILDING
    • Every entry has a type of building
    • If there are many types (ex: 800), we try to group them
    • Maybe we should separate the class cultural center from library
  4. BUILDING
    • Every entry has a building
    • See if it is convenient to extract the name in a new column (Ex: LIBRARY - PHILIPPINE ISLANDS, perhaps it is more interesting to keep two columns. One with LIBRARY - PHILIPPINE ISLANDS and another with PHILIPPINE ISLANDS)
  5. ADDRESS
    • Every entry has an adress (If they don't have it, look it up in google maps and fill in manually)

  6. DISTRICT

  7. DISTRICT CODE

  11. COORDINATE Y
    • Every entry has a coordinate Y (If they don't have it, look it up in google maps and fill in manually)

  12. LONGITUDE
